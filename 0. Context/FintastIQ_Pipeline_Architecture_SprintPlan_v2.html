Client Context Intake System

Capture business context before data processing — feeds smarter LLM decisions throughout the pipeline

Why Context Matters

Without context, the LLM guesses that "Bookings Dataset 1" is transaction data, that "TCV" means total contract value, and that the business is B2B SaaS. With context provided upfront, classification accuracy improves dramatically, entity resolution gets smarter, and imputation uses real business logic instead of statistical inference.

3-Tier Hybrid Approach

Tier 1 — Always Required (Structured)

Core Business Fields

5–6 fields that apply to every pricing engagement. Non-negotiable — pipeline won't start without them.

Company Name + Industry Vertical
Business Model B2B / B2C / Marketplace / Hybrid
Revenue Range Approximate annual revenue
Product/Service Type Products, services, subscriptions, mix
Engagement Goal One-liner on what they want
Tier 2 — Smart Follow-ups (LLM-Generated)

Dynamic Questions

3–5 questions generated by LLM based on Tier 1 answers. Tailored to the specific business type.

If B2B SaaS →
"What are your contract structures?" "Average deal cycle?" "Renewal rate?"
If Manufacturing →
"SKU count?" "Distribution channels?" "Raw material cost pass-through?"
If Professional Services →
"Hourly vs project-based?" "Utilization targets?" "Rate card structure?"
Claude Sonnet generates questions
Tier 3 — Open Context (Optional)

Free-Form + Deck Upload

Prompted open-ended fields plus the ability to upload proposal decks for automatic context extraction.

Pricing Structure "Describe in your own words..."
Known Issues "What's broken about pricing?"
Data Quirks "Anything weird we should know?"
Upload Deck/Doc .pptx / .docx → LLM extracts context

Who Fills This In?

Path A: FintastIQ Team (Internal Console)

Most common path. The team already has context from the sales process, case proposal, kickoff call. They fill in Tier 1 fields, or upload the proposal deck and let LLM extract everything automatically. They can also paste meeting notes or email threads.

Fill form manually Upload proposal .pptx Paste kickoff notes

Path B: Client Self-Service (Client Portal)

Optional path. Some clients may want to fill in context themselves when uploading data. The portal shows Tier 1 fields and Tier 3 prompts. Tier 2 follow-ups appear dynamically after Tier 1 is complete.

Guided form Dynamic follow-ups Optional context

Cross-Validation Step

After context is captured AND data is uploaded, the LLM cross-checks context against actual data. If client says "B2C retail" but data has contract_value and renewal_date columns, the system flags the mismatch and asks for clarification. The validated context becomes client_context.json — injected into every pipeline prompt.

LLM validates context vs data Each field tracks provenance (client-entered / team-entered / extracted / inferred)

client_context.json Schema

{
  "company_name": "NPI",
  "industry": "B2B SaaS / Consulting",
  "business_model": "B2B",
  "revenue_range": "$10M-$50M",
  "product_type": "Services + Subscriptions",
  "engagement_goal": "Pricing optimization, win rate analysis",
  "llm_follow_ups": { ... },
  "open_context": { ... },
  "provenance": {
    "company_name": "team-entered",
    "industry": "extracted-from-deck",
    "revenue_range": "inferred-from-data"
  },
  "validated": true,
  "validation_flags": []
}
